# H264码流基础学习笔记（从入门到能看懂hex）

---

# 一、H264本质是什么？

H264本质是：

> 一串 **NALU（Network Abstraction Layer Unit）**

结构：

```

NALU + NALU + NALU + ...

```

---

# 二、NALU结构（核心）

每个NALU：

```

[分隔方式] + [NALU头(1字节)] + [数据(RBSP)]

```

---

# 三、两种码流格式（必须掌握）

## 1️⃣ Annex-B（起始码模式）

```

00 00 00 01 + NALU
00 00 01    + NALU

```

特点：
- 用起始码分隔
- 常见于：
  - .h264裸流
  - RTSP / TS

---

## 2️⃣ AVCC（长度前缀模式）

```

[length(4字节)] + NALU
[length(4字节)] + NALU

```

特点：
- 没有 `00 00 00 01`
- 常见于：
  - MP4
  - 摄像头SDK

---

# 四、NALU头（重点！）

1字节结构：

```

| F | NRI | TYPE |

```

最重要：

```

nal_type = byte & 0x1F

```

---

# 五、常见NAL类型

| 类型(type) | 含义 |
|-----------|------|
| 1 | P帧（非IDR） |
| 5 | I帧（IDR，关键帧🔥） |
| 6 | SEI |
| 7 | SPS |
| 8 | PPS |

---

# 六、关键理解（非常重要）

## ❗ 不要死记 65 / 41

正确规则：

```

type = byte & 0x1F

```

---

## 示例：

| 字节 | 结果 | 类型 |
|------|------|------|
| 0x65 | 5 | IDR |
| 0x25 | 5 | IDR ✅ |
| 0x41 | 1 | P帧 |
| 0x01 | 1 | P帧 ✅ |

---

# 七、标准码流结构（真实）

```

00 00 00 01 67 → SPS
00 00 00 01 68 → PPS
00 00 00 01 25 → IDR（关键帧）
00 00 00 01 01 → P帧
00 00 00 01 01 → P帧
...

```

---

# 八、如何识别关键帧

## 方法：

```

byte & 0x1F == 5 → IDR帧

```

---

# 九、如何判断格式（Annex-B / AVCC）

## 方法1：

```

grep 00 00 00 01

```

- 有 → Annex-B
- 没有 → AVCC

---

## 方法2：

看hex：

### Annex-B：

```

00 00 00 01 67 ...

```

### AVCC：

```

00 00 02 34 67 ...

```

（前4字节是长度）

---

# 十、Annex-B解析流程（实战）

步骤：

1. 找 `00 00 00 01`
2. 后一个字节 = NALU头
3. 用 `& 0x1F` 判断类型

---

# 十一、GOP结构（帧序列）

典型：

```

I P P P P I P P ...

```

# 核心总结（一句话）

H264 =

```

(分隔符) + (NALU头) + (数据)

```

- Annex-B → 00 00 00 01
- AVCC → length
